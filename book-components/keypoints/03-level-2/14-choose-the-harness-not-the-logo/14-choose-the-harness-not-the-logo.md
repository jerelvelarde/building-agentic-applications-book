# Chapter 14 — Choose the Harness, Not the Logo

## Hook

Three tools can all edit a repository, yet one is a managed coding workflow, one an open provider-flexible harness, and one a persistent gateway across channels and devices. A feature checklist hides the decision.

## Reader outcome

Compare Claude Code, Hermes, and OpenClaw on dated engineering dimensions and select by use case, authority, and operating model.

## Core claims

- These systems overlap but are not interchangeable products.
- Compare primary shape, model posture, context/memory, tools/skills, permissions, isolation, MCP, delegation, recovery, observability, hosting, and trust model.
- Defaults and public APIs change; every row needs a date and primary evidence.
- Self-hosted does not automatically mean safer; managed does not remove application responsibility.
- There is no universal winner—only a fit for a defined boundary.

- **Primary table:** captured-version comparison from the Level 2 packet.
- **Decision cards:** “choose this when,” “you still own,” and “verify before deployment.”
- **Figure:** managed coding agent vs open harness vs persistent gateway topology.
- **Hands-on note:** only Claude CLI version was runtime-verified; no false benchmark.

## Exercise

Write a one-page decision record for a real workload: required environment, authority, models, isolation, persistence, integrations, team ownership, and recovery. Score products only after defining weights.

## Failure and security section

Call out stale defaults, one-dimensional ranking, subagents mistaken for OS isolation, container mistaken for complete security, and combining overlapping privileged terminal paths.
